# Como Estudar Esta Trilha

## Regra dos 3 passos

1. Leia o módulo.
2. Rode um exemplo.
3. Modifique algo e observe o resultado.

Se o código quebrou, ótimo: leia a mensagem de erro. Aprender Python passa por entender erros sem pânico.

## Use o lab de execução

Quando quiser testar uma ideia rapidamente, rode:

```powershell
python src\lab_execucao.py
```

Abra `http://127.0.0.1:8850`, escolha um exemplo e altere o código. A melhor prática é mudar uma linha por vez e executar de novo.

O lab bloqueia operações perigosas por padrão, limita leitura de dados à pasta `dados/` e só libera recursos ligados ao exemplo atual. Mesmo assim, use apenas para estudo local e evite colar códigos desconhecidos sem revisar.

## Diário de bordo

Crie um arquivo pessoal chamado `DIARIO.md` e registre:

- o que aprendi hoje;
- qual erro apareceu;
- como corrigi;
- qual conceito ainda está confuso;
- qual mini melhoria posso fazer amanhã.

## Checklist de cada módulo

- Consigo explicar o conceito sem olhar?
- Rodei pelo menos um exemplo?
- Fiz uma mudança por conta própria?
- Resolvi o desafio?
- Documentei o que aprendi?

## Como usar IA durante o estudo

Use IA para pedir explicações, revisar seu raciocínio e sugerir testes. Evite pedir a solução completa antes de tentar.

Um bom prompt:

```text
Estou aprendendo listas em Python. Tentei resolver este desafio e meu código falhou.
Explique o erro em linguagem simples e me dê uma dica, mas não resolva tudo.
```

# Visão Geral da Trilha

Esta trilha foi pensada como uma jornada de missões. Cada módulo apresenta um conceito, mostra exemplos pequenos e termina com um desafio prático.

## Mapa da jornada

| Etapa | Tema | Resultado esperado |
| --- | --- | --- |
| 00 | Boas-vindas e setup | Ambiente pronto e mentalidade de prática |
| 01 | Python base | Variáveis, entrada, saída, operadores e strings |
| 02 | Coleções e fluxo | Listas, tuplas, sets, dicionários, if, for e while |
| 03 | Funções e POO | Funções, classes, objetos, herança e encapsulamento |
| 04 | Arquivos e APIs | CSV, JSON, TXT, requests e organização de caminhos |
| 05 | Pandas e dados | DataFrames, limpeza, agregações e análise exploratória |
| 06 | Visualização | Gráficos para entender e comunicar dados |
| 07 | Banco de dados | SQLite, SQL básico e persistência |
| 08 | IA e ML | Regressão, classificação, métricas e anomalias |
| 09 | Projeto final | Assistente orientado a dados, inspirado pelo TaxPilot |

## Como a dificuldade aumenta

No início, os desafios são pequenos e quase todos rodam só com Python puro. Depois entram bibliotecas, arquivos reais, banco de dados e modelos de ML.

O foco não é decorar sintaxe. O foco é construir repertório: saber ler um problema, quebrar em partes, testar, explicar a solução e melhorar.

## O que evitar

- Copiar código sem rodar.
- Pular fundamentos para chegar logo em IA.
- Medir aprendizado apenas pelo tamanho do projeto.
- Usar modelos de IA sem entender entrada, saída, limites e métricas.

## Como apresentar no portfólio

Cada módulo pode gerar um pequeno commit ou registro de aprendizado. O projeto final deve conter:

- problema;
- dados usados;
- decisões técnicas;
- limitações;
- próximos passos;
- prints ou exemplos de execução.
